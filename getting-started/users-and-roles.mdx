# Users and Roles

Users access organisation workspaces through memberships and permissions.

## Membership

A membership connects a user to an organisation.

Membership may be active, suspended, or left depending on lifecycle state.

## Roles

Roles are used to group permissions.

Examples may include:

- Organisation admin
- Sales
- Operations
- Finance
- Support
- Platform admin

## Permissions

Permissions determine what a user can view or manage.

Examples include:

- `crm.customers.view`
- `quotes.manage`
- `bookings.convert`
- `invoices.issue`
- `payments.manage`
- `platform.organisations.manage`

## Platform admin access

Platform admins operate across organisations for system administration, support, subscription management, and platform settings.

Platform admin access is role and permission backed.
