# Core Concepts

Before using ArinnaOS, it is important to understand a few key concepts.

## Organisation

An organisation is a tenant within ArinnaOS.

All records belong to an organisation.

Examples:

- Travel agency
- Tour operator
- DMC

## User

Users access ArinnaOS through invitations.

Users belong to one or more organisations.

## Membership

Membership defines a user's relationship to an organisation.

Membership controls access rights.

## Roles

Roles define what a user can do.

Examples:

- Organisation Admin
- Sales
- Operations
- Finance

## Permissions

Permissions control specific actions.

Examples:

- Create customers
- Approve quotes
- Manage invoices

## Activity History

Important actions are recorded in activity logs for auditability.
