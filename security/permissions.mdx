# Permissions

ArinnaOS uses permission-based access control.

Roles group permissions, but permissions decide access.

## Rules

- Active organisation membership is required
- Matching organisation scope is required
- Platform admin override is controlled by platform permissions
- No `users.role` column is used

## Permission examples

- `crm.customers.view`
- `suppliers.manage`
- `tour_templates.publish`
- `quotes.accept`
- `bookings.convert`
- `invoices.issue`
- `payments.manage`
- `finance.reconcile`
- `platform.organisations.manage`
